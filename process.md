Set Up Local Storage & EncryptionTo keep the API key and question data secure on the user's device, use SQLite with an encryption layer.Encrypted API Storage: Use the keyring library to store the Gemini API key in the system's native secure storage (like Windows Credential Manager or macOS Keychain).Local Database: Use SQLAlchemy with SQLite to store "Classes" (Chapters/Types) and the extracted questions.Encryption: For sensitive data within the database, use the cryptography library to encrypt question content before saving it.2. PDF Processing & Question ExtractionGemini 1.5 Pro and Flash have native multimodal support, meaning they can "read" PDFs directly without needing complex OCR.Upload Logic: When a user uploads a PDF to a specific "Class," use the Google AI SDK for Python to send the file to Gemini.Prompting: Send a prompt like: "Extract every question from this PDF. Return them as a JSON list with 'question_text', 'options', and 'correct_answer' fields.".Parsing: Parse this JSON and save each question into your local SQLite database, linked to the specific Chapter/Class ID.3. Quiz Engine & TimerThe CLI interface will handle the quiz flow based on the user's selected parameters.Selection Logic: Query your local database for the chosen Chapter, then use a randomizer (like Python's random.sample) to pick the requested number of questions (10, 20, 30, or 50).Timer Implementation: Use Python’s threading or time module to track the duration.For 30 sec to 60 min, set a background thread that interrupts the CLI input when time expires.For No Time, simply loop through questions until the set is finished.4. Implementation StepsInitialize CLI: Create two main commands: config (to save the API key via keyring) and manage (to create chapters and upload PDFs).Process PDF: Use genai.upload_file() to send the PDF to Google’s servers temporarily for processing, then immediately delete it from the cloud after the questions are extracted and saved locally.Start Quiz: Create a start-quiz command that takes --chapter, --questions, and --time as arguments.Local Results: Store the user's score in the local database to track progress over time without needing an external account.
